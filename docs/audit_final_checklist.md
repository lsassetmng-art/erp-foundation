# ERP 監査（内部＋外部）最終チェックリスト

## A. 内部監査（Foundation AI / 内部監査）
- [ ] 監査は read-only（業務データを改変しない）
- [ ] 実行事実（実行/失敗/exit code）が記録される
- [ ] 承認証跡（PM承認）が残る
- [ ] 再実行は承認＋ポリシー一致のみ
- [ ] ループ防止（回数制限/同一監査1回制限）がある
- [ ] PM通知（Slack/LINE/File）が動作する
- [ ] 経営通知（KPI/週次/月次）が動作する
- [ ] 部門別KPI（管理会計）が生成される
- [ ] ログ保全・削除ルール（retention）が動作する
- [ ] 改ざんに弱い箇所がない（少なくとも protect フラグ運用が可能）

## B. 外部監査レイヤ（ERP機能 / External Audit Module）
- [ ] 入力は Foundation AI 出力のみ参照（業務テーブル直接参照禁止）
- [ ] 提出用レポート（月次/年次）を生成できる
- [ ] Evidence Pack（証跡パック）方針が定義されている
- [ ] 汎用語彙マッピング（internal→external）が定義されている
- [ ] 有効化はライセンス/ロールで制御可能（仕様がある）
- [ ] 出力は exports/ 配下に固定（監査人へ提出しやすい）

## C. 運用（CI/cron）
- [ ] CI 1行サマリが exit code 連動する
- [ ] cron/Actions のどちらでも回せる
- [ ] 失敗時の原因が監査ログから追える（理由コード or パターン）

## D. 変更管理
- [ ] 監査基盤は凍結（バグ修正以外はPM承認必須）
- [ ] 監査仕様が README / docs に固定されている
